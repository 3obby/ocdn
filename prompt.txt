OCDN Design System Overhaul — Sprint Prompts for Composer
==========================================================

Context for all sprints:
- Next.js 16 / React 19 app at /Users/dev/code/ocdn
- Dark-themed forum with two content layers: Bitcoin (permanent, gold) and Nostr/ephemeral (temporary, red)
- Tailwind v4 (no tailwind.config.js), shadcn/ui
- Test with: npx next build (user verifies manually)
- Do NOT create new documentation files. Minimize new files.
- Mobile-first. The shell is max-w-md centered.
- Preserve all existing functionality — these are visual/structural refactors only.

─── DESIGN LANGUAGE (applies to every sprint) ─────────────────────────────────

Typography:
  Geist Sans — all prose, labels, UI. Optimized for readability at small sizes.
  Geist Mono — ALL numbers: sats, view counts, pubkeys, confirmations, PoW scores,
    timestamps, countdowns. Numbers are data; data gets the mono treatment.
    Geist Mono is already bundled via the 'geist' package — import GeistMono
    from 'geist/font/mono' in layout.tsx alongside the existing GeistSans.
    Apply it as a CSS variable (--font-geist-mono) and use font-mono where needed.

Icons over text:
  Minimize labels. Icons communicate faster. Use lucide-react icons consistently:
    Eye (view count), Zap (boost/PoW), Pencil (reply), ExternalLink (on-chain),
    Plus (inscribe/upgrade to BTC), Clock (time/age), ChevronRight (expand).
  Never pair an icon with a text label if the icon alone is unambiguous.
  Numbers adjacent to icons are fine and expected (e.g., Eye + "829").

Color semantics:
  --accent-gold: #f4b63f   — Bitcoin / inscribed / permanent / durable
  --accent-red: #e93223    — ephemeral / temporary / expiring / danger
  --accent-burn: #f7931a   — sats amounts specifically
  --surface: #111111       — card/section background
  --bg: #000000            — root background
  --text-primary: #ffffff
  --text-muted: rgba(255,255,255,0.40)
  --text-ghost: rgba(255,255,255,0.15)
  --glow-gold: 0 0 4px rgba(244,182,63,.25), 0 0 10px rgba(244,182,63,.12)
  --glow-red: 0 0 4px rgba(233,50,35,.45), 0 0 10px rgba(233,50,35,.22)

Data layout — the spatial rule:
  DATA flows from the TOP-LEFT of every content cell.
  ACTIONS anchor to the BOTTOM-RIGHT of every content cell.
  Content text fills the middle.
  This creates a consistent diagonal reading path: metadata → content → actions.

Interaction buttons — the square rule:
  All interaction buttons are SQUARE (equal width and height), not rounded-full,
  not rectangular. Size: 32×32px default, 28×28px compact. Rounded corners: 6px
  (rounded-md). Background: transparent by default, bg-white/[0.06] on hover.
  Icon color: text-text-ghost idle, text-text-muted hover, accent color active.
  The REPLY button is always the bottom-right-most button, visually overlaid on
  the boundary/corner of the content area (positioned with negative margin or
  absolute positioning so it sits at the content edge). Boost and other actions
  sit adjacent to it (to its left), forming a horizontal row anchored bottom-right.

Compact = data-transparent:
  Show all relevant data, but use density — small font (10-11px), mono numbers,
  icon+number pairs, no wasted whitespace. Every pixel communicates something.
  If a piece of data exists, show it. Don't hide information behind clicks.
