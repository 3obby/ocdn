OCDN — MVP TASK LIST (zero to launch)

═══ SPRINT 1: PROTOCOL LIBRARY ═══

[ ] tagged_hash("ocdn/*", msg) — BIP-340 tagged hash helper
[ ] content_hash(pubkey, nonce, parent_ref, content) → 32 bytes
[ ] topic_hash(topic_string) → 32 bytes
[ ] sign_msg(version, type, content_hash) → 34-byte preimage
[ ] BIP-340 Schnorr sign/verify over sign_msg using @noble/curves
[ ] POST envelope serializer: tag + version + type + nonce + pubkey + sig + topic + content
[ ] REPLY envelope serializer: tag + version + type + nonce + pubkey + sig + parent_hash + content
[ ] BURN OP_RETURN serializer: tag + version + 0x03 + target_hash (38 bytes)
[ ] SIGNAL OP_RETURN serializer: tag + version + 0x06 + ordered refs (≤80 bytes)
[ ] envelope parser: detect "ocdn" tag, extract fields by type byte
[ ] OP_RETURN parser: detect "ocdn" prefix, parse BURN/SIGNAL payloads
[ ] content_hash verification: recompute from parsed fields, compare
[ ] author signature verification: recompute sign_msg, BIP-340 verify
[ ] unit tests: round-trip serialize→parse→verify for all four tx types
[ ] unit tests: invalid envelopes silently rejected

═══ SPRINT 2: BITCOIN INFRASTRUCTURE ═══

[ ] Bitcoin Core node setup (archival, unpruned, -txindex=1, -rest=1)
[ ] signet node for development/testing
[ ] RPC client wrapper: getblock (verbosity 2+), getblockcount, getbestblockhash
[ ] REST client wrapper: block iteration without JSON-RPC overhead
[ ] fee estimation wrapper: estimatesmartfee, mempool info
[ ] tx broadcast wrapper: sendrawtransaction, testmempoolaccept
[ ] block subscription: zmq or polling for new blocks

═══ SPRINT 3: WALLET + TX CONSTRUCTION ═══

[ ] portal keypair generation and secure storage
[ ] UTXO pool: track portal-owned UTXOs in DB
[ ] UTXO fan-out: pre-split into concurrent-ready outputs
[ ] UTXO consolidation: background defragmentation
[ ] Taproot script-path spend: envelope → commit tx (P2TR output with script tree)
[ ] Taproot reveal tx: spend commit output via script-path, envelope in witness
[ ] commit/reveal as package broadcast (1P1C relay)
[ ] key-path recovery on all commit outputs (portal internal key)
[ ] RBF signaling on all txs
[ ] fee bumping: RBF escalation for stuck txs
[ ] BURN tx construction: OP_RETURN + change
[ ] SIGNAL tx construction: OP_RETURN + change (self-signed mode)
[ ] retry pipeline: mempool eviction → rebroadcast → RBF bump → key-path recovery
[ ] cost estimator: vbyte calculation for each tx type at current feerate

═══ SPRINT 4: INDEXER ═══

[ ] PostgreSQL schema: posts, replies, topics, burns, signals, blocks, reorgs
[ ] block scanner: iterate blocks from genesis (or start height)
[ ] witness envelope detector: scan Taproot script-path spends for "ocdn" tag
[ ] multi-envelope support: parse ALL envelopes per witness, not just first
[ ] OP_RETURN scanner: detect "ocdn"-prefixed outputs
[ ] POST indexing: parse envelope, verify sig, store content + topic_hash
[ ] REPLY indexing: parse envelope, verify sig, validate parent exists, store
[ ] BURN indexing: parse OP_RETURN, compute tx fee (input values - output values), record
[ ] SIGNAL indexing: parse ref list, extract signer pubkey, compute fee, record
[ ] topic normalization: lowercase + trim + NFC for canonical topic hashes
[ ] thread tree builder: parent_hash chains → nested structure
[ ] ranking computation: total sats burned per content_hash and topic_hash
[ ] reorg handler: chain tip cursor, rewind on fork, re-process new chain
[ ] real-time mode: subscribe to new blocks, index incrementally
[ ] backfill mode: full historical scan
[ ] external protocol detection: recognize EternityWall and other known inscription formats
[ ] confirmation tracking: 0-conf display, 1-conf indicator, 6+ settled
