═══ 2. x402 AGENT API ═══

  Goal: AI agents pay sats to read/write durable data via HTTP.
  OCDN becomes persistent memory for autonomous agents.

  Steps:
    - Define x402 payment flow:
        Agent hits endpoint → gets 402 + Lightning invoice in header →
        Agent pays invoice → retries with payment preimage → access granted
    - Implement endpoints:
        POST /api/v1/inscribe   (write text to bitcoin, premium price)
        POST /api/v1/signal     (write signal, cheaper)
        GET  /api/v1/read/:hash (read content by hash, free or gated)
        GET  /api/v1/search     (query topics/content, free or gated)
        GET  /api/v1/feed       (ranked feed, free or gated)
    - Pricing tier: agent writes cost 2-5x human portal rake (premium
      for programmatic access to durable storage)
    - Auth: x402 payment preimage = access token (stateless, per-request)
    - Rate limiting by payment, not by IP
    - Response format: JSON, content + txid + block height + content_hash
    - Lightning integration: LND/CLN invoice generation + payment verification
    - Document the API as a simple spec agents can parse
      (or publish as an MCP tool descriptor)


═══ 3. EMBEDDABLE INSCRIPTIONS ═══

  Goal: every inscription has a shareable URL that renders a rich card
  on Twitter/social and works as an embed on blogs.

  Steps:
    - Route: /i/:contentHash (canonical URL per inscription)
    - OpenGraph + Twitter Card meta tags:
        og:title = first 80 chars of content
        og:description = "Permanently inscribed on Bitcoin · Block N · N sats committed"
        og:image = dynamically generated OG image (content text + block + sats)
    - OG image generation: Next.js og image route (satori/resvg or @vercel/og)
        Dark bg, gold accent, content text, block height, sats burned
    - oEmbed endpoint for rich embeds on platforms that support it
    - Copy-to-clipboard "share" button in the UI that copies /i/:contentHash URL
    - The inscription page itself: full PostCell expanded, with verify link
      to mempool.space txid

═══ 5. SEO + STATIC RENDERING ═══

  Goal: every inscription is indexable by search engines.

  Steps:

═══ 6. LIGHTNING PAYMENT INTEGRATION ═══

  Goal: users pay Lightning to inscribe (the "pay" step in ComposeSheet).

  Steps:
    - LND or CLN instance on VPS (or hosted: Voltage, Alby)
    - Invoice generation endpoint: amount = miner fee estimate + portal rake
    - Payment verification: subscribe to invoice settlement
    - On settlement: queue content for tx construction pipeline
    - Refund path: if tx construction fails after payment, credit or refund
    - Display: QR code in ComposeSheet pay step (already stubbed)
    - Fiat on-ramp: Stripe → sats conversion (or partner with existing on-ramp)

═══ 8. INDEXER (LIVE CHAIN SCANNING) ═══

  Steps:

    - Reorg handling: maintain chain tip cursor, rewind on fork
    - Backfill: scan from genesis (or first OCDN tx) on first run
    - Mempool monitoring: show unconfirmed txs with 0c indicator
