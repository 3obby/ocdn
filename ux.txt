SPRINT 6: PAYMENT GATE — TASK LIST
===================================

Current state: compose → submit → "PORTAL_PRIVKEY NOT SET". Full pipeline
proven on regtest. No user-facing payment flow. No free tier planned.

Model: every post starts as an ephemeral preview. Pay to make it durable (on-chain).
Unpaid posts optionally publish to Nostr as the "free" tier — visually weak,
indexed alongside durable content if found, upgradeable to on-chain at any time.

UX FLOW (mobile-first, 4 steps):
  1. compose   — type text, pick topic, tap send
  2. preview   — instantly cache + display as preview of durable state
                 subtle countdown: time remaining to pay before cache expires
  3. pay       — QR code (bitcoin: URI), copy address, "open in wallet"
                 poll for payment (0-conf mempool detection → seconds, not minutes)
  4. confirm   — payment detected → broadcast → "posted" + txid link

UNPAID PATH (nostr fallback):
  - on compose submit: immediately cache locally + display in feed as "ephemeral"
  - publish to nostr relay(s) as NIP-01 note (kind 1) with ocdn topic tag
  - ephemeral posts render visually weak: faded, smaller, no burn counter
  - indexer: scan nostr for ocdn-tagged notes, display alongside on-chain content
  - "upgrade to permanent" button on any ephemeral post → pay flow (step 3)
  - cache TTL: ~24h, then pruned from portal DB (still on nostr relays)

PAID PATH (on-chain, durable):
  - preview step shows cost breakdown: miner fee + rake = total sats
  - generate fresh taproot address per payment request
  - detect payment in mempool (0-conf), immediately queue broadcast
  - portal constructs commit/reveal, broadcasts, records PendingTx
  - received payment UTXO enters portal pool → self-sustaining
