═══ YOUR TASK: SPRINT 3 — WALLET + TX CONSTRUCTION ═══

build the Bitcoin transaction construction pipeline. this is the code that takes a signed OCDN envelope (from the protocol library) and turns it into real Bitcoin transactions that can be broadcast to the network.

the tasks from tasks.txt:
  [ ] portal keypair generation and secure storage
  [ ] UTXO pool: track portal-owned UTXOs in DB
  [ ] UTXO fan-out: pre-split into concurrent-ready outputs
  [ ] UTXO consolidation: background defragmentation
  [ ] Taproot script-path spend: envelope → commit tx (P2TR output with script tree)
  [ ] Taproot reveal tx: spend commit output via script-path, envelope in witness
  [ ] commit/reveal as package broadcast (1P1C relay)
  [ ] key-path recovery on all commit outputs (portal internal key)
  [ ] RBF signaling on all txs
  [ ] fee bumping: RBF escalation for stuck txs
  [ ] BURN tx construction: OP_RETURN + change
  [ ] SIGNAL tx construction: OP_RETURN + change (self-signed mode)
  [ ] retry pipeline: mempool eviction → rebroadcast → RBF bump → key-path recovery
  [ ] cost estimator: vbyte calculation for each tx type at current feerate

key details from vision.txt you need:
  - POST and REPLY use Taproot script-path spends with commit/reveal tx pairs
  - the envelope goes inside an OP_FALSE OP_IF ... OP_ENDIF script in the Taproot script tree
  - the portal's internal key is always the Taproot internal key (for key-path recovery)
  - commit tx creates a P2TR output; reveal tx spends it via script-path, placing envelope in witness
  - Bitcoin Core 28+ supports 1-parent-1-child package relay (commit + reveal broadcast together)
  - BURN is a single OP_RETURN tx (no commit/reveal needed)
  - SIGNAL is a single OP_RETURN tx (self-signed mode: the Bitcoin UTXO signer IS the author)
  - RBF signaling (sequence < 0xfffffffe) on ALL transactions
  - key-path recovery: if a commit confirms but reveal gets stuck, portal can recover funds via key-path spend
  - the Utxo and PendingTx tables in the Prisma schema are already designed for this

use bitcoinjs-lib (npm: bitcoinjs-lib) for Taproot/PSBT construction — install it. you may also need @cmdcode/tapscript or tiny-secp256k1; use whatever is most appropriate.

put the wallet/tx code in src/lib/bitcoin/wallet.ts and src/lib/bitcoin/tx.ts (or split further as makes sense). write tests.

to test, we use 'npx next build'; no need to run the dev server. the prisma database has valuable live data, please append-only to .env.
